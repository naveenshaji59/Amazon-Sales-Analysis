Amazon Sales Performance Analysis (Week 2 Internship Project)
📋 Project Overview
This project focuses on analyzing a large-scale retail dataset from Amazon (128,000+ transactions) to help a business understand its sales performance and optimize its warehouse operations. By transforming raw data into a professional Excel dashboard, I identified the best-selling products, top-performing regions, and significant "revenue leaks" caused by order cancellations.

🗺️ Project Roadmap
Data Import: Loading the raw CSV into Microsoft Excel.

Data Cleaning: Fixing dates, handling missing prices, and cleaning "noisy" data.

EDA (Phase 3): Using formulas to find the core health metrics of the business.

Data Visualization: Building charts to "see" trends in categories and states.

Deeper Insights: Using Pivot Tables to cross-reference product categories with size demand.

Strategic Strategy: Turning data findings into actionable business advice.

🎯 Project Objectives
Identify Revenue Drivers: Which product categories (e.g., Sets, Kurtas) generate the most money?

Regional Analysis: Which states in India represent our strongest customer base?

Inventory Optimization: Which clothing sizes (M, L, XL, etc.) are in highest demand?

Operational Health: What is the success rate of our orders, and how can we reduce cancellations?

📊 Dataset Understanding
Total Records: 128,975 rows.

Key Columns: Order ID, Date, Status, Category, Size, Amount, and Ship-State.

Initial Condition: The raw data contained missing amounts, inconsistent date formats, and scientific notation errors in IDs.

🛠️ Phase-by-Phase Process
Phase 1 & 2: Raw Data Processing
Before analysis, I "scrubbed" the data to ensure accuracy:

Null Values: Identified blank cells in the Amount column and filled them with 0 (representing unsuccessful transactions).

Date Standardization: Converted varied text-based dates into a unified DD-MM-YYYY format.

Data Integrity: Fixed scientific notation errors in the Order ID and Postal Code columns.

Cleanup: Removed empty columns to keep the dataset light and efficient.

Phase 3: Exploratory Data Analysis (EDA)
I calculated the "Big Numbers" using Excel formulas:

Total Revenue: ₹78.6 Million

Total Quantity Sold: 116,649 units

Average Order Value (AOV): ₹661

Total Transactions: 120,378 orders

Phase 4: Data Visualization
I built a professional dashboard containing:

Category Analysis: A bar chart showing that "Sets" and "Kurtas" are the primary revenue drivers.

Fulfillment Status: A pie chart revealing a 14.2% cancellation rate.

Geographic Sales: A bar chart highlighting Maharashtra and Karnataka as the top markets.

Phase 5 & 6: Deeper Insights (Pivot Tables)
Using Pivot Tables, I analyzed Category vs. Size demand.

Finding: Sizes M, L, and XL are the most popular across all top categories. This insight helps the warehouse prioritize stock for these sizes.

💡 Key Insights & Business Recommendations
Inventory Strategy: Focus 60% of warehouse space on Size M and L for the "Set" and "Kurta" categories.

Marketing Focus: Increase advertising spend in Maharashtra and Karnataka, as these regions show the highest demand.

Revenue Protection: Investigate the 14.2% cancellation rate. Reducing this by even 3% could recover millions in potential sales revenue.

Pricing Strategy: Since the Average Order Value is ₹661, the business should introduce "Bundle Offers" to push the average spend higher.

🏁 Conclusion
This project successfully moved from raw data to a strategic roadmap. By cleaning the data and using professional analysis tools in Excel, we now know exactly what to stock, where to sell, and how to protect the company's revenue.
